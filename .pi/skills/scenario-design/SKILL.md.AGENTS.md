# scenario-design/SKILL.md — index

Drafts adversarial real-life test scenarios from OpenSpec change spec. Core = (input · trigger · observable) Triple; unfillable slot → spec gap. Phase 1 classify requirements by shape → ISTQB technique (EP+BVA, decision table, state-transition, convergence, perf, fault injection). Phase 3 Gate: HARD (proposal/design) calls `ask_user` + STOP, SOFT (apply) annotates `[NEEDS CLARIFICATION]`. Phase 4 routes scenarios to L1 unit / L2 qa smoke / L3 Playwright e2e, plus an additive `manual-only` outcome (aesthetics/hardware/subjective — no fold, level —); L3 harness port is the dynamic `.pi-test-harness.json` `dashboardPort`, not `:18000`. Phase 5 writes `openspec/changes/<name>/test-plan.md` with a mandatory `disposition` column (`automated`\|`manual-only`) per row — the manifest is source of truth for automated-vs-manual. Never invents missing values, never writes test code. See change: add-openspec-pipeline-orchestrators.
