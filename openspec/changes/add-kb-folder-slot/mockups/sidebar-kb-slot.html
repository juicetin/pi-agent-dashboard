KB folder slot — sidebar-folder-section

New row per folder, sibling of Goals / Automations. Count on the left, action on the right. Mock — states shown side by side.

What the row shows

  • Count — store.counts().chunks (tooltip adds files). One glance = "is this folder's KB populated?"
  • ↻ Reindex — POST /api/kb/reindex?cwd=… → existing indexSource. Incremental (mtime→sha256), so a no-op is cheap.
  • → arrow — (optional) opens a KB detail/search overlay for the folder, mirroring the goals board route.

State machine

  • populated — chunks > 0, fresh.
  • empty · worktree — chunks === 0. Primary case for a fresh worktree that didn't inherit the parent db → prominent Index now.
  • indexing — spinner + optional file-progress bar while the walk runs.
  • stale — reads dox-staleness.json (already tracked): N files changed since last index.
  • error — route failed → Retry.

Why worktrees are the killer case

loadConfig(cwd) resolves a per-cwd dbAbsPath. A worktree checked out at a new path has no .pi/dashboard/kb/*.db → 0 chunks → kb_search there returns nothing until indexed. Today that's invisible; this slot makes it a one-click fix.
