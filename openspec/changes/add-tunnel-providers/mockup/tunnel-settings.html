Pairing QR — TLS https/wss only (D14 intact); runs the crypto handshake, issues a bearer over TLS. A MagicDNS name with tailscale cert qualifies. Link QR — no-TLS http endpoints; encodes the bare URL, opens the dashboard directly, no pairing payload / no crypto.subtle / no bearer over the wire. Arrival access via trustedNetworks; mesh underlay (WireGuard) encrypts transport. Corrected after doubt review: http can't run browser pairing crypto (secure-context rule) and would leak the bearer on a non-enclave mesh.

Security: the button runs a fixed whitelisted recipe keyed by (provider, step) — never a free-form command. The pasted key is validated, posted over the authenticated loopback path, written to the provider's own config (~/.zrok2, ngrok.yml), and never logged.

Public (funnel) mode adds two gated steps: enable HTTPS certs (console link), and funnel in ACLs — each blocks the public URL until detected.
