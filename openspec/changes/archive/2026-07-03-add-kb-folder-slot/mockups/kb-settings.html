How it maps to the backend

  • Opens from the KB folder row's → at route /folder/:cwd/kb (same overlay pattern as the goals board).
  • Sources / Include / Exclude / DB path = the editable paths. Everything else in KbConfig (ranking, chunking…) is round-tripped untouched.
  • Read GET /api/kb/config?cwd → { config, origin }. Save PUT /api/kb/config?cwd → validateConfig → atomic write → optional reindex.
  • origin pill: project = has file · global/defaults = falling back → shows Copy from parent / Create project config.
  • Worktree fix: Copy from parent repo reads the parent's knowledge_base.json, rewrites sources[] relative to the worktree cwd, and PUTs — so the folder-slot reindex button then has something to index.
  • Security: cwd validated ∈ known folders; source paths escaping the folder warn (KB reads md only, never executes).
