Mockup for add-session-uncommitted-indicator-and-commit — indicator states · commit dialog (file picker + AI draft) · mobile

1 · Indicator states on the session card (GIT subcard)

The dirty/drift pill sits on the existing GitInfo line after branch · #PR · worktree. Whole pill is a button → opens the commit dialog. A Commit button joins the GIT subcard. Hidden entirely when the tree is clean and in sync.

1b · Multiple sessions in the SAME directory (not a worktree)

These sessions share one working tree, so the shell already suppresses the per-card git block (showGitInfo = group.length === 1) and shows one shared GroupGitInfo line in the folder header. The dirty pill + Commit live there, at folder level — one tree, one commit. The child cards below carry no git block.

1c · Switch branch dialog (existing control — opened by ⌘ Switch branch)

Clicking Switch branch opens the existing BranchSwitchDialog. Shown for context, and because its Uncommitted Changes step is where this proposal ties in: today it only offers Stash & Switch — with the commit feature, Commit… becomes a first-class alternative (commit, then switch clean).

2 · Commit dialog — file picker + AI-draft lifecycle

Pick a subset of changed files, then write the message by hand or let the AI draft it from the session's own context (fork-subagent — no conversation pollution). Draft is always editable before commit.
