2 · Proposed — one severity scale, five tiers

One token vocabulary shared by toasts, banners, and status dots. Severity ⇒ color, always.

CRITICAL
error
--accent-red #ef4444
Operation failed. Needs action. Spawn crash, save failure.
WARNING
warning
--accent-orange #f97316
Degraded / caution. Retry pending, partial result.
SUCCESS
success
--accent-green #22c55e
Completed. Spawn started, committed, saved.
INFO
info / notice
--accent-blue #3b82f6
Neutral fact. Hidden card, reasoning-only, seek hint.
NEUTRAL
muted
--text-muted #6b6b6b
Passive / ended. No severity.

3 · Migration shape (no code here — explore only)

StepChange
1Extend ToastVariant = error | warning | success | info; add a warning entry to VARIANT_CLASSES.
2Flip showToast default from error → info (neutral), so unmarked = harmless.
3Tag success call sites: spawn-success + Committed … ⇒ "success".
4Re-point toast info from gray → --status-notice (blue) so "info" is one color.
5Replace raw red-900/green-900 in Toast + SpawnErrorToastHost with the shared --status-* / severity tokens.
6Introduce a single --severity-{critical,warning,success,info,neutral} token set; map both status + toast to it.
