## Why

`kb dox lint` reports 9 residual `over-threshold` issues. All are the row-count arm (`> ROW_CAP 40`), not the byte arm. Two very different shapes hide under the one `over-threshold` kind, and the lint conflates them:

1. **Rollup directories** — an `AGENTS.md` that documents files living in *subdirectories* that were never given their own `AGENTS.md`. `qa/AGENTS.md` has 54 rows but only **3** files are at `qa/` root; the other 51 belong to `packer/` (17), `tests/` (26), `fixtures/` (4), `scripts/` (4). `docker/AGENTS.md` has 47 rows but 30 belong to `docker/fixtures/`. This is a straight violation of the DOX "one dir = one `AGENTS.md`" model, fixable with **zero source churn** — scaffold the missing subdirectory `AGENTS.md` and move rows down.

2. **Genuinely flat source directories** — every file sits at one level: `packages/client/src/components` (175 files, 172 rows), `packages/server/src` (140, 136), `packages/client/src/lib` (119, 111), plus the marginal `packages/client/src/hooks` (50, 47), `packages/extension/src` (49, 47), `packages/shared/src` (49, 44), and `tests/e2e` (62 specs). Reducing rows here means physically foldering source into cohesive subfolders and updating imports.

There is also a policy gap. The **byte cap** (`AGENTS_BYTE_CAP` 30 KB) is the real per-turn context cost — pi auto-injects a directory `AGENTS.md` on every turn — and it is already solved by the file-based sidecar split (`scripts/split-large-agents.mjs` promotes heavy rows to pull-only `<File>.AGENTS.md`). The **row cap** (40) is an advisory proxy that does not, on its own, bound injection cost once a file is under the byte cap. Today a row-over-only flag reads as unfixed debt when it is often benign. The lint should say which arm tripped, so "reach zero over-threshold" stops being a mechanical goal and becomes a judgment about directory cohesion.

## What Changes

- **Fix the `ROW_CAP` count to measure inline detail, not file count** (`packages/kb/src/dox.ts`): the over-threshold row check SHALL count only **inline** rows and exclude promoted **sidecar-pointer** rows (a row whose purpose carries the `→ see `<File>.AGENTS.md`` marker left by `scripts/split-large-agents.mjs`). A pointer row is pull-only and cheap, so promoting a heavy row to its sidecar SHALL reduce the counted total. This makes the metric measure inline-detail volume rather than raw file count. `ROW_CAP` stays `40` (the number is not raised); only *what is counted* changes.
- **Split `over-threshold` severity in `kb dox lint`**: emit distinct signals for the **byte** arm (actionable — file-split remedy) and the **row-only** arm (informational/advisory — no injection cost once under the byte cap). Row-over is evaluated on the fixed inline count from the bullet above.
- **Decompose rollup directories (doc-only, no source moves):** scaffold per-subdirectory `AGENTS.md` for `qa/` (`packer/`, `tests/`, `fixtures/`, `scripts/`) and `docker/` (`fixtures/`, `scripts/`), moving each row down to the `AGENTS.md` that owns it. Clears `qa/` + `docker/` immediately.
- **Fold the three oversized root-level source directories into cohesive domain subfolders**, each a separately-shippable increment: `packages/client/src/components/` (~172 root-level files; already has 11 subfolders — `DirectorySettings/`, `Gateway/`, `chat/`, `editor-pane/`, `extension-ui/`, `interactive-renderers/`, `preview/`, `split/`, `tags/`, `tool-renderers/`, `__tests__/`), `packages/server/src/` (~138; already has `browser-handlers/`, `lib/`, `model-proxy/`, `routes/`, `rpc-keeper/`, `test-support/`, `tunnel-providers/`), `packages/client/src/lib/` (~118; only `__tests__/`). Behavior-preserving: relocate **root-level** files into subfolders — **absorbing into an existing subfolder where one already fits** (e.g. the root `MarkdownPreview*`/`Image*` peers of the existing `preview/` files go into `preview/`), creating a new subfolder only for a domain with no existing home; then update **all** references and give each new subfolder its own `AGENTS.md`. Moved files are owned by their subfolder's `AGENTS.md` (the tree is walked, not rolled up) — the parent does **not** gain per-subfolder pointer rows; its row count drops as files leave.
- **Explicitly accept the marginal directories as informational** under the new severity split: `hooks/` (~47 rows), `extension/src/` (~47), `shared/src/` (~44), `tests/e2e/` (~63 specs). No source churn for barely-over counts unless a natural cohesive grouping emerges during the big-three work. (`extension/src/AGENTS.md` sits at ~96% of the byte cap — the "informational-only" classification is a current-state fact, not a permanent guarantee: if any marginal dir later crosses the byte cap, its actionable byte-arm remedy — sidecar split — applies.)

## Capabilities

### New Capabilities
- `dox-directory-foldering`: `kb dox lint` counts only inline rows toward `ROW_CAP` (sidecar-pointer rows excluded, via a dedicated inline counter that leaves `parseRowPaths` path semantics untouched) and distinguishes the byte-cap arm (actionable) from the row-cap arm (informational) of `over-threshold`; the DOX tree is decomposed so no directory's `AGENTS.md` rolls up files from un-scaffolded subdirectories, and the oversized root-level source directories are foldered into cohesive per-domain subfolders (absorbing into existing subfolders where they fit, each subfolder owning its files via its own `AGENTS.md`), behavior-preserving.

## Impact

- **Code:** `packages/kb/src/dox.ts` (inline-row counter + `doxLint` over-threshold severity split) + rebuild `packages/kb/dist`. `parseRowPaths` is a public export consumed by `packages/kb-extension/src/reindex.ts` (`acknowledgeRows`/`decideNudge`) — the inline-count fix MUST add a sibling counter and NOT change `parseRowPaths`'s signature, or kb-extension breaks; rebuild kb-extension too. File moves under `packages/client/src/components`, `packages/server/src`, `packages/client/src/lib`, with repo-wide reference updates: not only ESM import specifiers (preserving the `.js`-on-`.tsx` convention) but also **string-literal path references** — several `packages/shared/src/__tests__/no-*.test.ts` allowlists name source files as `"packages/.../<file>.ts"` strings that `tsc` will not flag; they must be grep-updated or tests fail at runtime.
- **Docs / tree:** new subdirectory `AGENTS.md` across `qa/`, `docker/`, and every new source subfolder; parent `AGENTS.md` rows collapse to per-subfolder pointer rows. Follows the existing Documentation Update Protocol (per-directory `AGENTS.md` tree); no root-`AGENTS.md` rows added.
- **Extends, does not reverse:** builds on the DOX tooling shipped by `markdown-knowledge-base` §6d and the `dox-source-tree-migration` change. `ROW_CAP` stays 40; only *what it counts* (inline rows) and its *severity classification* change.
- **Staging / reversibility:** the lint fix (inline count + severity split) lands first (so residual marginal dirs read correctly), then each directory is an independent increment guarded by `tsc --noEmit` + `npm test` and a `kb dox lint` row-count check. Increments are independently **shippable**; a clean isolated **revert** holds only when no later fold re-touched the same importer files (folds of `components/`/`server/src/`/`lib/` share importers) — see design D5.
- **Out of scope:** raising `ROW_CAP` numerically; foldering the marginal 44–47-row dirs and `tests/e2e`; the `doxEnforcement` write-hook; any behavioral change to the app itself.
