# WorktreeInitButton.tsx — index

Hook-run-only amber Initialize button per directory/worktree row. Renders button iff `hasHook===true && (trusted===false || needsInit===true)`, fail-open hides on error. Click → `POST /api/git/worktree/init`; untrusted → ConfirmDialog naming gate + run; on confirm re-issues with confirmHash. Streams `worktree_init_progress` to live tail via worktree-init-bus; `worktree_init_failed` → spawn-error-style failure card; success re-fetches init-status → button disappears. See change: generalize-worktree-init-hook. Migrated to client-utils Dialog/Confirm shell. See change: unify-dialog-system. Probe source: props `status?`/`onStatusChange?` — when `status !== undefined` the row owns the shared single probe (`useInitStatus`) and this button neither self-fetches nor self-refetches; when `undefined` it self-probes + self-refetches (standalone/tests). The polymorphic no-hook scaffold branch (`onInitializeProject`, `project-init-btn`) was extracted to `ProjectInitButton`; this button is now strictly the hook runner. See change: distinguish-initialize-actions.
