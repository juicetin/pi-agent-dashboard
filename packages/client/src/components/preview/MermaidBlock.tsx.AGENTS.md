# MermaidBlock.tsx — index

Renders fenced mermaid blocks via lazy `mermaid.render()`. Module-level `_svgCache`/`_errorCache` keyed code+`themeId` (`<themeName>:<resolved>`) so per-named-theme accents cache separately; avoids re-render flicker; serialized `renderQueue` guards concurrent renders. `sanitizeMermaidCode` (entity decode + dedent), `sanitizeMermaidSvg` (strip script/on*). `renderMermaid` sanitizes then colorizes once before caching. `colorizeDefaultNodes(svg, accents, textColor)` DOMParser pass tints default (un-authored) flowchart/class nodes with theme accent palette: fill=accent@8% wash, border=accent@85%, label keeps `--text-primary`; skips shapes whose inline `style` has `fill:` (author intent wins); color keyed by `hashId(g.id)` (djb2, edit-stable). `resolveAccents` reads 6 `--accent-*` vars live (fallback ramp for jsdom); `rgba(hex,alpha)` helper. Streaming-aware via `complete` prop; focus-gated zoom/pan + `ZoomControls`. Exports `MermaidBlock`, `_svgCache`, `_errorCache`, `colorizeDefaultNodes`, `hashId`, `rgba`. See change: colorize-mermaid-default-nodes.

NOT migrated to the shared escape-stack (inline focused mode, no portal/backdrop; sibling diagrams focus independently, so LIFO would track focus not z-order). Focused-mode Escape double-close is a deferred follow-up handled with local focus-scoped Escape, not the modal stack. See change: fix-stacked-escape-closes-layers (Eligibility rule / Non-goals).
