# SessionBanner.tsx — index

Single-card error-lifecycle surface, driven by `deriveBannerState`. ONE bordered card: error string is the header (or `retry.reason` when retry-only, no settled error yet); provider auto-retry is a live sub-line `retrying… (attempt N)` (`data-testid="retry-banner"`) + a thin animated top strip (`data-testid="retry-strip"`) while retrying. Never two stacked cards. Props: `state`, `onAbort`, `onDismiss`, `now?`, `collapseThreshold?` — NO `onRetry`. ✕ (`error-banner-dismiss`) is CLEAR-ONLY: calls `onDismiss` only, NEVER `onAbort`. "Stop (ends the session)" (`error-banner-stop`) is the sole abort (`onAbort`), rendered only while retrying. No manual "Try again". No `limit-exceeded`/💳 variant — billing/quota renders as ordinary error. Long messages collapse via `error-banner-toggle` (Show more/less). Card testid `error-banner`; text `error-banner-text`; attempt `retry-banner-attempt`. Mounted sticky between `ChatView` and `CommandInput`. Imports `BannerState` from event-reducer. See change: simplify-error-retry-single-card.
