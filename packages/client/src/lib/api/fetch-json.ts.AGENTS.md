# fetch-json.ts — index

Shared client transport guard. Exports `ApiHttpError` class (`status`, `statusText`, `contentType`, `bodySnippet` ~200 chars) + `fetchJson<T>` + `fetchJsonResponse<T>`. `fetchJson` validates `res.ok` AND content-type includes `application/json` before parse; else throws `ApiHttpError` message `HTTP <status> <statusText>`. Use for helpers that throw on failure. `fetchJsonResponse` applies content-type guard only (not `res.ok`); returns `{ res, json }`. Use for status-branching callers (checkout 409, worktree-create error union) that read JSON from non-2xx responses. Closes non-JSON-response bug: proxy/gateway HTML error page no longer surfaces as `Unexpected token '<'`. Migrated callers: git-api.ts, browse-api.ts, editor-api.ts, known-servers-api.ts, providers-api.ts, tools-api.ts. See change: guard-client-fetch-json.
