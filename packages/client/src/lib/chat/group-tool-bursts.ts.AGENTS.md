# group-tool-bursts.ts — index

Temporal burst grouping — OUTER pass over `groupConsecutiveToolCalls`. Exports `ToolBurstGroup {type:"burst";id;items}`, `BurstItem = ChatItem\|ToolBurstGroup`, `groupToolBursts(messages)`. Composition FLIPPED to semantic-INNER-first: runs `groupConsecutiveToolCalls` over the FULL stream FIRST, then `burstWindow` helper walks the `ChatItem[]`. Burst = maximal run of tool-like items (a `toolResult` row OR a `×N` group = ONE member) across transparent items (`thinking`/`turnSeparator`/`rawEvent`/`commandFeedback`/EMPTY `assistant`); HARD boundary = `user`/non-empty `assistant`/`interactiveUi`/`bashOutput`/`inlineTerminal`. UNIVERSAL grouping (threshold 1): forms on ≥1 member; TURN-SCOPED window absorbs BOTH leading transparents (buffered in `pending` until a tool-like confirms the run) AND trailing transparents (`burstWindow`) up to the surrounding HARD row, so opening + concluding reasoning fold INSIDE. Exception (`emitToolRun`): a lone `×N` group stays bare when its absorbed transparents are all STRUCTURAL (rawEvent/turnSeparator/commandFeedback/empty-assistant) — they emit standalone around it; a lone `×N` that absorbed real `thinking` DOES wrap so reasoning folds in (no double-frame otherwise). Identical calls split by prose fold into a nested `×N`. `id` = first tool-like member id (group→`messages[0].id`; React key, survives event-trim). See changes: group-tool-call-bursts, collapse-tool-calls-across-narration, enhance-tool-call-grouping.
