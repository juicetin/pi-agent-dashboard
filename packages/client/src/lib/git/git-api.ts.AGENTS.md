# git-api.ts — index

Client git/worktree API helpers for BranchPicker + WorktreeSpawnDialog. Exports `fetchBranches`, `checkoutBranch` (returns `CheckoutResult` discriminated union, 409→dirty), `gitInit`, `stashPop`, `fetchGitHead`, `fetchWorktrees`, `createWorktree` (union w/ error `code` + `orphanLikely`), `fetchWorktreeInitStatus`, `fetchAutoInitWorktreePref`, `setAutoInitWorktreePref`, `runWorktreeInit` (TOFU confirmHash flow), `probePathExists`, `cleanupOrphanWorktreePath`, `removeWorktree`, `mergeWorktree`, `pushWorktreeBranch`, `createWorktreePR`, `fetchPullRequests`, `createWorktreeFromPr`, `fetchWorktreeDiffStat`. Mirrors server `WorktreeInitHook` type. `WorktreeInitStatus` gains `configured?: boolean` (present only on `hasHook:false`; distinguishes unconfigured dir ① from configured-no-hook ③; absent on fail-open). `fetchActiveInits()` → `ActiveWorktreeInit[]` (GET /api/git/worktree/active-inits; fail-open `[]`) for boot rehydration. See changes: add-worktree-spawn-dialog, generalize-worktree-init-hook, add-worktree-lifecycle-actions, add-worktree-from-pull-request, distinguish-initialize-actions, friendlier-worktree-init. Adds `fetchGitStatus` (GET /api/git/status; null on failure), `fetchChangedFiles` (GET /api/git/changed-files), `commitFiles` (POST /api/git/commit → `CommitResult` union), `draftCommitMessage` (POST /api/git/commit-draft; empty message → manual entry). See change: add-session-uncommitted-indicator-and-commit.
