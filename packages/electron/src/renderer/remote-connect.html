Connect to Remote Dashboard

Point this app at a dashboard running on another machine (for example a Docker host or a server on your network) instead of starting a local one.

ⓘ On the server machine, the dashboard must be started with network access enabled (listening on the network, not just localhost), and the OS firewall must allow the port. Otherwise it won’t be reachable from here.

Connecting restarts PI Dashboard so it can attach to the remote server. Your local server (if running) keeps running.
