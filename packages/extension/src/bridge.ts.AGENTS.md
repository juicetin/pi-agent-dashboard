Wires `PromptBus`, `DashboardDefaultAdapter`, provider register, role manager, flow listeners, metrics monitor, model/git/session trackers. `tool_execution_end` runs `inlineToolResultImages`; applies its result on result-identity change (not `inlinedCount>0`) so strip-only rewrites propagate. See change: fix-duplicate-inlined-screenshot. On `agent_end`, classifies terminal assistant msg via `classifyTurnActionability` + `EmptyActionableGuard.observe`: `continue` → `enqueueSystemFollowup(CONTINUATION_NUDGE)` (only when `bridgeFollowUp` empty); `surface` → forwards `empty_actionable_surface` event (non-error). Never leaves an empty-actionable turn silent-idle. See change: fix-gemini-subagent-silent-tool-schema-failure. Routes `role_remove` (emit `roles:remove`; on success re-emit `roles_list`); all `roles_list` sends carry `builtinRoleNames`. See change: add-custom-roles-ui. See change: simplify-error-retry-single-card — removed the usage-limit auto-abort + `UsageLimitOrderer` + `USAGE_LIMIT_PATTERN` import. Retry synthesis is observe-based: assistant `message_start` drives `retryTracker.observeMessageStart` (emits `auto_retry_start`); `message_end` drives `observeMessageEnd` (error records pending, non-error closes chain); `agent_end` drives `observeAgentEnd`. No billing special-casing. `pi.events.emit` intercept treats subagent channels as reconcilable: forward live only when `sessionReady && isActive() && connection.isConnected`, else `subagentFrameBuffer.buffer` (covers `!sessionReady` AND transient WS drops — `sessionReady` stays true across reconnects, so gating on `connection.isConnected` keeps reconnect-window frames in the per-agent buffer instead of the shared FIFO ring). `flushPendingSubagentFrames()` after `sessionReady=true` AND in `onReconnect`; `subagentFrameBuffer.reset()` on session change/shutdown. `onMessage` handles `subagent_resync_request` → replies synthetic `subagents:started` `event_forward` from `subagentFrameBuffer.resync`. Registers `session_info_changed` → `autoNamer.onObservedName` (push external renames early; poll fallback) and `project_trust` → `decideProjectTrust` (auto-trust dashboard-spawned headless in `activationCwd`, captured at activation).
