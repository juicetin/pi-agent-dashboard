# role-manager.ts — index

Manages session model roles. Registers six `roles:*` handlers (`roles:get-all`/`set`/`remove`/`preset-load`/`preset-save`/`preset-delete`) — renamed from `flow:role-*`, NO alias (change: add-agent-role-model-tools D11). `roles:get-all` payload adds `builtinRoleNames = [...DEFAULT_ROLE_NAMES]`; `roles:set`+`roles:remove` re-validate name via shared `isValidRoleName(role, [])`; `roles:remove` rejects built-ins server-side then `removeRoleFromSchema`+save (change: add-custom-roles-ui). Exports `DEFAULT_ROLE_NAMES`, `lookupRole(ref)` (single `@role`→literal accessor: `{literal?,reason?}`, strips `@`, re-reads disk), `getModelRole` (delegates to `lookupRole`), `overlayDefaultRoles`, `overlayRoles(cfg)`/`effectiveRoleNames(cfg)` (effective schema = defaults ∪ added − removed), `addRoleName`/`removeRoleFromSchema` (mutate cfg; purge-on-remove). `RoleConfig` gains `roleNames`/`removedRoles`. `roles:get-all` overlays effective schema at read time — never empty, no auto-write. `role:resolve-model` = deprecated one-release alias via `lookupRole`; sets `probe.reason` when unconfigured.
