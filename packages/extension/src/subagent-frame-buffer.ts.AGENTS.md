# subagent-frame-buffer.ts — index

Pure class `SubagentFrameBuffer` + `SUBAGENT_CHANNELS` set. Makes running-subagent timeline reconcilable. D1: `buffer(channel,data)` retains latest frame per `agentId` while bridge not ready (bounded `maxAgents`=64, drop-oldest, re-insert=most-recent); `drain()` returns buffered frames emission-order + clears (bridge flushes on re-register). D2: tracks latest snapshot per RUNNING agent (dropped on `subagents:completed`/`failed`), BOUNDED to `maxAgents` (same latest-wins/drop-oldest as `pending` via `evictToBound`) so resync storage stays bounded; `resync(agentId)` returns it or undefined (unknown/finished no-op). `markForwarded` counts ready-path + updates snapshot. `reset()` on session change/shutdown. `stats` counters (forwarded/buffered/flushed/droppedNoAgentId/overflowEvicted/resync*). Statics `isSubagentChannel`, `agentIdOf`. See change: fix-subagent-live-detail-reliability.
