Strips consumed path from text (no dead link, D5). Over-cap/missing/non-image left as text → falls back to artifact route. Pure; `readFile` injected. Adds `isAllowedPath` gate (artifact-root containment) before disk read; preserves pre-existing non-text content blocks; padding-aware base64 byte budget. Byte-dedup: skips appending an image whose bytes match an image block already in the result (e.g. MCP `browser` returns text + native image for same screenshot) → strips path, no side-by-side duplicate; genuinely-different images still inline.
