# tui-prompt-adapter.ts — index

Production PromptBus adapter for Pi's TUI. Exports `createTuiPromptAdapter(ui, bus)` and `TuiPromptUi`. Presents select/input/confirm/editor through captured unpatched `ctx.ui` methods (registered before `ctx.ui` is patched). Forwards confirm `metadata.message` as the rendered second arg (empty string when absent) so terminal confirm dialogs keep the context the dashboard already shows. Boolean confirm → `"true"`/`"false"` string; null answer → cancelled. Aborts its dialog when another adapter answers first or on cancel via a per-prompt `AbortController` map. Intentionally has no multiselect arm — pi 0.70 RPC mode's `ctx.ui.custom` is a no-op and would auto-cancel the dashboard-rendered dialog (see change: fix-multiselect-tui-arm-self-cancel); multiselect is handled by the bus-routed `ctx.ui.multiselect` patch + `DashboardDefaultAdapter`. See change: fix-confirm-context-in-pi-tui (PR #286).
