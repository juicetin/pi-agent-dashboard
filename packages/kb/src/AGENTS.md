| `dox.ts` | DOX tree. Directory-level AGENTS.md scaffold + audit. kb agents <path> nearest-applicable chain. Detect-don't-write: dox init/--fix fill PATH columns + prune orphans only. `over-threshold` lint splits two arms: `arm:"bytes"` (actionable — bytes > `AGENTS_BYTE_CAP` 30 KB, auto-injected per turn, remedy sidecar split) + `arm:"rows"` (informational — inline count > `ROW_CAP` 40, no per-turn cost, optional foldering). `countInlineRows(af)` counts INLINE rows only (excludes sidecar-pointer rows matching `/→ see \`<File>.AGENTS.md\`/`) — sibling to `parseRowPaths` (public export consumed by kb-extension; signature untouched, stays complete path list). Splitting a heavy row to its sidecar drops both byte + counted-row totals. `ROW_CAP`/`AGENTS_BYTE_CAP` numeric values unchanged. `isMdFile` excludes `*.AGENTS.md` sidecars + `*.agent.md` companions (both DOX index artifacts, not documentable source). `DEFAULT_EXCLUDE` skips build output (`out`) + electron bundled/vendored trees (`bundled-extensions`, `electron/resources/server`) — all gitignored, fs-walk-only, else bogus missing/companion rows; `server` scoped to `electron/resources/server` so real `server` source dirs stay indexed. `resolveRowPath(agentsDir,cwd,rp)` resolves rows relative to their OWN AGENTS.md dir, repo-root `cwd` fallback (nested AGENTS.md may document a file living at project root) before orphan. `parseRowPaths` + `doxLint` scan only rows under a `# DOX —` heading (prose tables like Subagent Routing ignored). `doxLint` `walkAgents` tests `DEFAULT_EXCLUDE` on the cwd-RELATIVE path (mirrors `walkFiles`) so running inside `.worktrees/` does not exclude the whole tree. `rowPaths`/staleness keyed cwd-relative. See change: split-components-agents-dedup-rollup. See change: fix-dox-lint-false-positives. See change: fold-oversized-agents-directories. |
| `eval.ts` | Retrieval-quality eval. Scores search against golden set. Gates ranking changes. |
| `index-run.ts` | Atomic `kb index` orchestration. `runIndexAtomic({dbPath,sources,indexOpts,explicit})` returns `IndexStats & {counts}` (counts read from the still-open store so `index` never re-`openStore`s). First index (no dbPath) builds into `<dbPath>.tmp-<pid>` + `rename()` on success (survives OOM/SIGKILL — real path only on success); incremental (dbPath exists) indexes in place (mid-run failure leaves prior DB valid & queryable). All-sources-missing + explicit-`--source`-missing throw BEFORE any store opens (no husk). `sweepOrphanTemps` unlinks stale `.tmp-*` but SKIPS a temp whose PID is a live process (concurrent peer's active build). See change: harden-kb-index-failure-atomicity. |
| `index.ts` | Public API barrel for @blackbelt-technology/pi-dashboard-kb. |
| `indexer.ts` | Indexer. `indexSource` is ASYNC: walks source, mtime→sha256 change detection, structural chunking, Tier-1 graph extraction, BATCHED transactional upsert — commits + `await setImmediate` every `YIELD_EVERY`=100 files so the walk yields the event loop (a concurrent `/stats` reader observes `indexing`+live counts) and releases the WAL write lock; trades whole-walk atomicity for responsiveness (mid-walk throw leaves earlier batches committed; reindex idempotent). All callers must `await`. A source whose `dir` does not exist is skipped with `console.warn` (returns `missing:true`), not a thrown ENOENT — partial source set still indexes what exists. `docTypeOf` classifies `*.AGENTS.md` per-file index sidecars as `agents` doc_type (searchable; name != `AGENTS.md` so pi never auto-injects them). See change: split-components-agents-dedup-rollup; fix-kb-index-feedback; harden-kb-index-failure-atomicity. |
| `init.ts` | kb init. Scaffolds + validates knowledge_base.json. --global writes global file. --force, --dry-run flags. gitignores dbPath. |
| `migrate-file-index.ts` | One-time migration core. Re-homes `docs/file-index-<area>.md` purposes into per-dir `AGENTS.md` tree. Pure + testable. Exports `parseFileIndex`, `mergeIndex`, `planDirs`, `tier0Rows`, `renderAgentsMd`, `validateAuthored`, `finalRows`, types `IndexRow`, `FileEntry`, `DirPlan`, `AuthoredRow`. Classifies dirs tier 0 (all hits, deterministic) vs tier 1 (≥1 miss, needs subagent). |
| `migrate-runner.ts` | Big-bang resumable runner for file-index → AGENTS.md migration (one-time; source splits since deleted). Owns plan, batching, grounding gate, idempotent per-dir write, checkpoint persistence. Exports `loadSplitTexts`, `buildDirPlans`, `groundingCheck`, `knownStems`, `makeBatches`, `subagentPrompt`, `parseAuthoredBatch`, `recordAuthored`, `writeDir`, `loadState`, `saveState`, `PACKAGES_ROOT_RE`. Drives `@fast` subagent fan-out for Tier-1 authoring. Rollup export (`exportRollup`/`treeRows`/`SPLIT_AREAS`) REMOVED — `docs/file-index*.md` retired, per-directory `AGENTS.md` tree is sole per-file record. See change: split-components-agents-dedup-rollup. See change: migrate-file-index-to-agents-tree. |

See change: fix-kb-index-feedback; harden-kb-index-failure-atomicity. |
