| File | Purpose |
|------|---------|
| `directory-handler.ts` | Browser→server handlers for pin/unpin/reorder dirs, favorite/unfavorite models, folder-workspaces CRUD + reorder, session reorder, openspec refresh + bulk archive, extension_ui_response forward, pi-gateway forwards (request_commands/list_files/request_models/request_providers/set_thinking_level/set_model). Uses `canonicalizePath` (normalize then realpath) before storage. |
| `handler-context.ts` | Defines `BrowserHandlerContext` interface — shared handler dependency bag passed to every browser-message handler. Carries `ws`, `sessionManager`, `eventStore`, `piGateway`, `headlessPidRegistry`, `pendingResumeRegistry`, optional `preferencesStore`, `sessionOrderManager`, `directoryService`, `terminalManager`, `metaPersistence`, pending registries (fork/attach/worktree-base/resume-intent/client-correlations), `viewMessageStore`, send/broadcast/replay callbacks. |
| `session-action-handler.ts` | Browser→server session action handlers: `handleSendPrompt` (auto-resume ended sessions), `handleResumeSession` (continue/fork, placement intent, empty-session fork degrade), `handleSpawnSession` (preflight + watchdog arm + spawn-failure logging; enqueues optional `initialPrompt` into `pendingInitialPromptRegistry` — see change: project-init-skill-and-profiles), `handleShutdown`, `handleAbort`, `handleStopAfterTurn`, `handleForceKill`, follow-up queue forwarders (`handleClearFollowupEntries` … `handlePromoteFollowupEntry`), `handleFlowControl`, `handleKillProcess`, `handleHeadlessReload` (kill+respawn reload for headless `/reload`). Exports `FORK_DEGRADED_TO_NEW_MESSAGE`, `FORK_DEGRADED_TO_NEW_CODE`. |
| `session-action-helpers.ts` | Pure helpers for `session-action-handler.ts`, extracted for unit tests. Exports `shouldInterceptReload` — returns true when `send_prompt` text is exactly `/reload`, no images, and session PID tracked in `headlessPidRegistry.getPid(sessionId)`. |
| `session-meta-handler.ts` | Browser→server session metadata handlers: `handleRenameSession`, `handleHideSession`, `handleUnhideSession` (moveToFront ordering), `handleAttachProposal`/`applyAttachProposal` (idempotent attach + auto-rename), `handleAcceptReplaceProposal`/`handleDismissReplaceProposal`, `handleDetachProposal`, `handleSetSessionDisplayPrefs`, `handleSetSessionProcessDrawer`, `handleFetchContent`, `handleListSessions`. Exports `pushAttachProposalChanged` (pushes `attach_proposal_changed` to bridge). |
| `subscription-handler.ts` | Browser→server subscription handlers: `handleSubscribe` (replay stored events in batches with backpressure, stale-lastSeq reset + full replay, on-disk load via `directoryService`, asset/UI-state replay) and exports `replayUiState`, `replaySessionAssets`, `sendEventBatches`. Replays `ui_modules_list`, `ui_data_list`, `ext_ui_decorator`, `plugin_intents`, view-messages snapshot. Cold-load branch emits a hydration heartbeat (`setInterval` `HYDRATE_HEARTBEAT_MS=10000`) re-sending the empty `event_replay{events:[],isLast:false}` marker to live subscribers while `loadSessionEvents` parses; `stopHeartbeat()` clears it in every exit path (success/cancelled/error/catch), guarded by `ws.readyState===OPEN`. Keeps the client's hydration ceiling alive on very large parses. See change: fix-history-loading-false-empty-flash. |
| `terminal-handler.ts` | Browser→server terminal handlers: `handleCreateTerminal` (spawn PTY + reorder), `handleOpenInlineTerminal`/`handleCloseInlineTerminal` (event-sourced inline terminal card via `inline_terminal_open`/`inline_terminal_close` events), `handleKillTerminal`, `handleRenameTerminal`. |
