# goal-store.ts — index

Per-cwd GoalRecord persistence. `GoalCreateBody`/`GoalUpdateBody` gain `judge?`+`autoRespawn?`. create/update persist judge+autoRespawn. `appendVerdict(cwd,id,verdict)` pushes GoalVerdict, FIFO-caps at `GOAL_VERDICTS_CAP`. `applyStatus(cwd,id,projection)` sets durable `status`+`lastKnownTurnsUsed`+`totalTurnsUsed` (adds non-negative `turnsDelta` cumulatively under mutex)+`lastProgressAt`. Supervisor methods: `listAll()` (enumerate all folder files, boot reconcile), `replaceDriver(cwd,id,newSessionId)` (swap driver even when set + capture `currentDriverBaselineTurns=totalTurnsUsed`), `recordRespawn(cwd,id,GoalRespawn)` (FIFO `GOAL_RESPAWNS_CAP`), `setStatus(cwd,id,status,reason?)`, `finalize(cwd,id,{status,statusReason?,clearInFlightSpawn?})` (bump `generation`+terminal status in ONE write — abort ordering), `setInFlightSpawn(cwd,id,GoalInFlightSpawn\|null)`. `GoalStatusProjection`/`GoalFinalizeBody` types. Legacy GoalsFile loads unchanged (new fields optional). See changes: sophisticate-goal-authoring-and-control, persist-goal-status-and-progress, add-goal-session-supervisor.
