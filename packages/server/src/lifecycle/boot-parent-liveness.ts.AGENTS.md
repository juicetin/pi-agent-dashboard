# boot-parent-liveness.ts — index

Boot-parent liveness + live-ppid reader for `/api/health`. `bootParentPid` const (captured once at module load = `process.ppid` at boot). `computeBootParentAlive(): boolean` — two-tier: Tier 1 `isProcessAlive(bootParentPid)` (all platforms, PID-reuse-vulnerable); Tier 2 (win32 only, OPTIONAL `koffi` dep) holds a `SYNCHRONIZE` `OpenProcess` handle + `WaitForSingleObject(h,0)` (PID-reuse-safe), jiti-safe `createRequire` load in try/catch, permanent Tier-1 fallback on any failure, never throws. `readLivePpid(): number` — reparenting-aware live ppid (Linux `/proc/self/stat` field 4; macOS `ps -o ppid=`; Windows `process.ppid`); NOT cached `process.ppid`. `bootParentLivenessTier(): "tier1"\|"tier2"` — diagnostic reporting the ACTIVE tier (tier2 = koffi handle loaded + OpenProcess succeeded, win32 only); asserted by `scripts/windows-liveness-smoke.ts`.
