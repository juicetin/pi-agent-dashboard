# recovery-server.ts — index

Pure `node:http` recovery server. `startRecoveryServer({port, error})` spawned by `cli.ts` `runForeground` when top-level deps (fastify/toad-cache/etc) fail to resolve. Binds same port. Serves status HTML at `/`, `GET /api/health` returns `{mode:"recovery", error}`, `POST /api/recovery/retry` respawns dashboard via `process.execPath`, `POST /api/recovery/reinstall` runs `npm install` in detected layout. Zero third-party imports. `parseModuleNotFoundError(msg)` extracts missing specifier; `isModuleNotFoundError(err)` checks `ERR_MODULE_NOT_FOUND` / `MODULE_NOT_FOUND`; `detectInstallLayout(cwd)` returns `monorepo` / `npm-global` / `electron-bundled` / `unknown`; `suggestedReinstallCommand(layout)` maps layout → shell cmd; `buildRecoveryHtml({error, layout, cmd})` escapes XSS. Writes `~/.pi/dashboard/last-recovery.json` snapshot (timestamp, error, layout, cmd) on start. See change: add-startup-recovery-server.
