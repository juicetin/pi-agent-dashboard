# DOX — packages/server/src/model-proxy

Files in this directory. One row per source file.

| File | Purpose |
|------|---------|
| `api-key-store.ts` | Pure proxy API-key helpers: `hashKey` (sha256 hex), `verifyKey` (constant-time), `generateKey` (`pi-proxy-<48 base64url>`), `findApiKey` (discriminated `FindResult`: valid/revoked/expired/miss), `recordKeyUsage` (60s-debounced lastUsedAt), `keyHasScope` (`"all"` matches any). Types `FindResult`, `ProxyScope`. |
| `auth-gate.ts` | Fastify `onRequest` hook factory `createModelProxyAuthGate` for `/v1/*` proxy routes. Extracts bearer token, enforces `pi-proxy-` prefix, applies `FailedAuthBackoff` delay, calls `findApiKey`/`keyHasScope`/`recordKeyUsage`, attaches `proxyApiKeyId`/`proxyApiKeyLabel` to request. Returns 401/403 codes `AUTH_REQUIRED`, `AUTH_MALFORMED`, `PROXY_KEY_REQUIRED`, `AUTH_REVOKED`, `AUTH_EXPIRED`, `SCOPE_INSUFFICIENT`. |
| `concurrency.ts` | Nested concurrency limiter `ConcurrencyTracker.acquire({apiKeyId, provider}, config)` returns release() callback; throws `ConcurrencyError` with codes `SERVER_FULL`/`KEY_FULL`/`PROVIDER_FULL` + `retryAfterMs`. Tracks server-wide, per-key, per-provider counts; reads caps from `ModelProxyConfig` at acquire time for live config updates. |
| `failed-auth-backoff.ts` | Per-source-IP exponential backoff for failed proxy auth. Class `FailedAuthBackoff`: `record(ip)` doubles delay from 10ms cap 10s, `reset(ip)` on success, `getDelayMs(ip)` peeks. In-memory only. |
| `custom-provider-discovery.ts` | Server-side custom-provider discovery (Approach C). Exports `discoverProviderModels(name, entry)` + `discoverAllCustomProviders(providers)`: for each `providers.json#providers` entry resolves apiKey (`resolveProbeApiKey`) + lists `/v1/models` (`listProviderModelIds`), returns `CustomModelEntry[]` carrying provider `baseUrl`/`api` + api-typed fallback capability floors. Never throws (unreachable provider → []). NO models.json. See change: add-agent-role-model-tools. |
| `internal-auth-storage.ts` | Server-resident auth storage `InternalAuthStorage`. Reads `~/.pi/agent/auth.json` via `provider-auth-storage.ts`, returns API key + headers per model. Constructor takes optional `customProviderCreds` resolver — custom-provider api_key creds (from providers.json, not auth.json) fall back here so custom-provider models route (change: add-agent-role-model-tools). For OAuth credentials, `ensureFreshOAuth` refreshes tokens 30s before expiry (provider-specific `getOAuthProvider` then generic `refreshOAuthToken` fallback), serializes concurrent refreshes per provider in `refreshLocks`, persists via `writeCredential`. `OAUTH_PROVIDER_MAP` maps anthropic/openai-codex/github-copilot. |
| `internal-registry.ts` | Server-resident model registry `InternalRegistry`. Composes pi-ai built-in providers with custom-provider models discovered from `providers.json` (`discover()` — fills the former no-op loop via `deps.discoverCustomProviders`, cached) + any user-authored `models.json`, filters to authed models via `canRouteModel` (api_key routes all; oauth routes only `oauthCompatible !== false`). `refresh()` re-runs discovery. Exports `getAvailable`, `find`, `firstAvailable(preferred[])` (walk ordered fqids, first available in registry; skips bare labels; null on empty/none), `getApiKeyAndHeaders`, `refresh`, `discover`, `getAll`, `getAllAnnotated` (excludedReason: no-credential/oauth-incompatible). `getAllModels()` dedups by fqid `provider/id` keeping first (precedence built-in → discovered-custom → models.json) — at most one entry per fqid, `find` deterministic. Types `PiAiModule`, `CustomProviderEntry`, `CustomModelEntry`, `InternalRegistryDeps` (+ optional `discoverCustomProviders`). See change: add-agent-role-model-tools, fix-and-prefer-model-proxy-resolution. |
| `oauth-compat.ts` | OAuth-incompatible model override table. Exports `OAUTH_INCOMPATIBLE: Record<provider, ReadonlySet<modelId>>` + `isOauthIncompatible(provider, id)` (unknown provider/id → false; case-sensitive). Flags legacy Anthropic snapshots (`claude-3.x` dated ids) unreachable over OAuth. `internal-registry.ts` sets `oauthCompatible = !isOauthIncompatible(...)` on built-in models. Hand-maintained; review when provider ships new model. See change: filter-oauth-incompatible-models. |
| `recursion-guard.ts` | Recursion guard preventing custom providers pointing back at dashboard. `collectDashboardOrigins(port, opts)` enumerates localhost + LAN IPs + optional tunnel/mDNS hostnames. `isSelfPointing(baseUrl, origins)` normalizes URL host:port and checks against origin list. |
| `registry-singleton.ts` | Singleton accessor for `InternalRegistry`. `getModelRegistry()` lazily resolves pi-ai via `getDefaultRegistry().resolveModule`, loads oauth subpath, constructs `InternalAuthStorage` + `InternalRegistry` with disk readers for `~/.pi/agent/providers.json` and `~/.pi/agent/models.json`, wires `discoverCustomProviders: discoverAllCustomProviders` + fires initial `discover()`. `readAugmentedAuth` merges auth.json with synthetic custom-provider api_key creds (`readCustomProviderCreds`) so discovered custom models pass `getAvailable`'s auth filter. Also exports `refreshModelRegistry`, `disposeModelRegistry`, `getStreamSimpleFn`, `getModelProxyStatus` (ready/degraded). See change: add-agent-role-model-tools. |
| `request-log.ts` | Optional JSONL request logger `logRequest(entry)` to `~/.pi/dashboard/model-proxy.jsonl`. Logs metadata only (no bodies, no keys): `RequestLogEntry` carries ts, requestId, apiKeyId, model, format, status, durationMs, tokens, error. Rotates at 50MB to dated suffix. Best-effort, never throws. |
| `streamer.ts` | Wrapper `streamCompletion(opts, piAiStreamSimple, registryOverride?)` around pi-ai `streamSimple`. Resolves API key + headers from registry (`getModelRegistry` default), assembles context (messages/systemPrompt/tools) + options (apiKey/headers/maxTokens/temperature/signal), returns `AsyncIterable` of pi-ai StreamEvents. Types `StreamCompletionOpts`, `RegistryLike`. |
