# pairing.ts — index

QR/copy-string pairing manager (D6/D12). `PairingManager({registry,getFingerprint,getReachableUrls,now?})`: `createPayload()` mints ~60s one-time code + payload `{v,id,code,urls[]}` (secure origins only, D14); `redeem(code)` creates ONE pending device + 8-digit confirm code (does NOT consume code); `approve(code,typedConfirmCode,label?)` compare-and-match then consumes code + mints bearer (rate-limit/lockout); `poll(pendingId)` one-shot token collect. `reachableUrls()` admits only https/wss (D14) EXCEPT a loopback http origin (`localhost`/`127.0.0.1`) when `PI_E2E_SEED=1` (`isTestLoopbackOrigin`; localhost is a browser secure context) so the Playwright/Docker harness runs the real handshake without TLS — never true in prod. Exports `PAIRING_PROTOCOL_VERSION`, `SUPPORTED_PAIRING_VERSIONS`. See change: add-server-keypair-pairing. See change: make-pairing-qr-camera-scannable.
