# memory-event-store.ts — index

In-memory event store with LRU eviction; replaces SQLite-backed event-store. Exports `StoredEvent`, `EventStore`, `createMemoryEventStore(isSessionPinned, maxCachedSessions, maxEventsPerSession, maxStringFieldSize, maxEventDataSize)`, `exceedsSerializedSize(value, cap)`, `DEFAULT_MAX_CACHED_SESSIONS`, `DEFAULT_MAX_EVENTS_PER_SESSION`, `DEFAULT_MAX_EVENT_DATA_SIZE`. Per-session buffers, truncates large string fields, `findToolEndEvent` reverse-scans for `tool_execution_end`. Skips pinned sessions in eviction. Per-session trim preserves chat head: drops oldest NON-essential event first (tool/subagent/flow/reasoning noise), only drops oldest `message_start`/`message_end` under extreme pressure; `DEFAULT_MAX_EVENTS_PER_SESSION`=20000 (was 5000, subagent floods parent buffer). Per-event total-serialized-size ceiling (`DEFAULT_MAX_EVENT_DATA_SIZE`=20000, `0`=off): after per-field truncation, `exceedsSerializedSize` (bounded early-exit walk, never full-stringifies) gates replacing over-cap `event.data` with a `{__truncated}` placeholder — stops one oversized subagent event OOM-ing the broadcast `JSON.stringify`; `truncateStrings` depth>4 now truncates/collapses to `[truncated: deep]` (preserves image blocks) instead of returning raw. See change: bound-subagent-event-serialization. See change: preserve-chat-head-on-event-trim. `capString` skill-envelope-aware: `SKILL_ENVELOPE_RE` truncates BODY only, keeps `<skill>` header/closing tag/args intact so client `parseSkillBlock` survives; base64 image blocks exempt from per-event size ceiling. See change: bound-subagent-event-serialization (skill regression). Store-shed telemetry: `trimBufferToLimit` returns `{dropped,toolEndDropped}` (counts `tool_execution_end` among drops in the same O(n) pass), `evictIfNeeded` returns evicted-buffer count; closure accumulates `trimmedEventsTotal`/`trimmedToolEndTotal`/`trimmedEventsBySession`(Map)/`evictedSessionsTotal`; `getTrimStats()` handle returns `TrimStats {trimmedEvents:{total,toolExecutionEnd,bySession},evictedSessions}` (global counters cumulative/never-reset; `bySession` lifecycle-scoped — entry dropped in `evictIfNeeded`/`deleteEventsForSession` when its buffer is removed, so the Map cannot accumulate stale sessions). See change: instrument-event-store-trim.
