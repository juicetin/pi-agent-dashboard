`extractFileChanges(events, cwd)` scans `tool_execution_start` write/edit events, groups by path, attaches preceding assistant message as context → `FileDiffEntry[]`. `enrichWithGitDiff(cwd, files)` computes Changed Files diff via git → adds `isGitRepo`. `gitNumstat(cwd)` parses `git diff --numstat` (tab-separated adds/dels/path; binary `-` omitted) → per-path count map; `enrichWithGitDiff` attaches per-file `additions`/`deletions` + `totalAdditions`/`totalDeletions`. `enrichWithVcsDiff(cwd, files)` → `VcsEnrichmentResult` dispatcher wraps `enrichWithGitDiff`, annotates `vcsKind`/`diffBase`/`baseLabel` (optional fields older clients ignore). See change: add-change-summary-table.

**Async + event-loop-safe (fix-session-diff-eventloop-block):** `enrichWithGitDiff`, `enrichWithVcsDiff`, `gitNumstat`, `buildSessionDiff` are all `async`; every git spawn on the `/api/session-diff` path is non-blocking (`runAsync` via `git.isGitRepoOrAsync`/`numstatOrAsync`/`diffAllOr`/`statusPorcelainOrAsync`/`headShaOrAsync`) — NO `spawnSync` git reachable from the handler. Per-file content diffs come from ONE batched `git diff --relative HEAD` (`git.diffAllOr`) split per file by `splitBatchedDiff(raw)` → `Map<path, chunk>` (splits on `diff --git` header boundaries; NEW path from `+++ b/` → `rename to ` → header `b/` fallback), replacing the old O(files) per-file `git.diffOr` loop. Tracked chunks over `TRACKED_DIFF_MAX_BYTES` (5 MB) or binary (`isRenderableTrackedDiff`/`isBinaryDiffChunk`) listed with numstat counts but no `gitDiff` (992 MB `.tar` never text-rendered). Untracked new files keep the synthetic `readFileSync` path (git diff never lists untracked). `buildSessionDiff(events, cwd, opts?)` accepts pre-fetched `{ gitRepo, porcelainRaw }` to avoid re-spawning. `buildSessionDiffCached(sessionId, events, cwd, cache)` computes key `sessionId:HEAD-sha:djb2(porcelain)` then defers to `SessionDiffCache` (TTL hit + single-flight coalesce). See change: fix-session-diff-eventloop-block.

`buildSessionDiff(events, cwd)` = full orchestrator behind `/api/session-diff`: (1) `extractFileChanges`; (2) DETECT on-disk files — git `parsePorcelain(git.statusPorcelainOr, cwd)` (C-unquote, rename→new, skip deletions, `normalizePath` shared key space, out-of-cwd dropped) OR non-git `parseBashArtifacts` + in-cwd `existsSync`; (3) ATTRIBUTE via `parseBashArtifacts` (`bashOutputCandidates`: `>`/`>>`/`-o`/`--output`/`tee`; `redactCommand` strips secrets + caps 120; last-writer-wins); (4) COMPOSE precedence — write/edit kept, `mixed` when also detected, NO synthetic ghost event, detector-only → one `type:"tool"` event; (5) OWNERSHIP gate — write/edit ∨ Bash-token ∨ mtime ∈ `extractBashWindows` (±1s slack) → `files` (`sessionOwned:true`), else `otherChanges[]`; (6) cap `MAX_FILES=200` (write/edit precedence) then enrich. `enrichWithGitDiff(cwd, files, {untracked})` threads the bulk-porcelain untracked set (no per-file re-probe) + binary/size safety (`isBinaryFile` ext/NUL-sniff, `SYNTHETIC_DIFF_MAX_BYTES=256KB`) BEFORE any utf-8 read → binary/oversized rows listed with no `gitDiff`. See change: detect-tool-created-files.
