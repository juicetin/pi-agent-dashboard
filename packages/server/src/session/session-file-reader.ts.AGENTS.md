# session-file-reader.ts — index

Standalone JSONL session reader. Exports `SessionEntry`, `loadSessionEntries(filePath)` (leaf→root branch order via parentId), `createBranchedSessionFile(sessionFilePath, targetEntryId)` for fork-from-message. Linear fallback when no tree structure. Exports `findSessionToolCallPayload(filePath, toolCallId)` → `{ content?, edits? } | null`: scans `message.content[]` for `{ type:"toolCall", id === toolCallId }` (nested id, not entry top-level), returns `arguments.content` (Write) / `arguments.edits` (Edit) from the on-disk JSONL — the FULL untruncated payload surviving the in-memory ~4 KB cap / >20-op collapse. SECURITY: takes only a transcript path + a toolCallId used for equality (no path input, no traversal); missing file / unknown id / unparseable → null, no path fallback. Backs `GET /api/session-change`. See change: opt-in-out-of-cwd-session-diffs.
