# session-load-worker-pool.ts — index

Session-load worker pool. Fixed slots = `max(1, min(maxConcurrentSpawns, os.cpus().length))`; FIFO queue when slots busy; per-request timeout default `30000ms`; lazy spawn per slot via `new Worker(url, {execArgv: process.execArgv})` so jiti hook propagates. `load(req)` returns `{jobId, result}`. `cancel(jobId)`: queued job dropped, resolves `"cancelled"`; in-flight job abandoned, result discarded on arrival, worker NOT terminated (only timeout/crash terminates slot). Fallback: spawn fail / crash / non-zero exit / timeout → in-process `loadAndReplay`; `result` never rejects. `useLoadWorker=false` → always in-process, no spawn. Copies `openspec-poll-worker-pool` scaffold, not extracted. See change: offload-session-events-load-to-worker.
