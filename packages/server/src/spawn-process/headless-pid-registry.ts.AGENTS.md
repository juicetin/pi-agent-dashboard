# headless-pid-registry.ts — index

Registry mapping headless child processes → session IDs. Exports `createHeadlessPidRegistry`, `HeadlessPidRegistry`, `keeperOptsFromSpawnResult`, `KeeperRegisterOptions`, `KeeperWriter`. Three-tier link (`linkByToken`/`linkByPid`/`linkSession`), keeper-mode `piPid` vs `keeperPid` split, `killBySessionId` (SIGTERM→2s→SIGKILL; kill pi first so keeper auto-exits), `writeRpc` (UDS forward), `cleanupOrphans`/`cleanupKeeperOrphans` (disk-persisted `spawnToken`/`piPid`/`keeperPid`/`keeperSockPath` survive restart). 7-day orphan max age. Test-guarded. `killByToken(spawnToken)` kills a spawned-but-unlinked entry (no sessionId) by stored token; shares the `killEntry` ladder with `killBySessionId`. `register(...,goalId?)` stamps an optional `goalId` on the entry (keyed to spawn token, round-trips restart); `getGoalId(sessionId)` resolves it after `linkByToken` — the PRIMARY goal-driver correlation (replaces goal cwd-FIFO).
