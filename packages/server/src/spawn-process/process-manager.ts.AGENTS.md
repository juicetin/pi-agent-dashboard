# process-manager.ts — index

Spawns/kills pi sessions. Exports `spawnPiSession`, `buildSpawnEnv`, `buildHeadlessArgs`, `buildInteractivePiArgs`, `buildTmuxCommand`, `shellEscape`, `setResolver`/`resetResolver`, `setKeeperManager`/`getKeeperManager`, `SessionOptions`, `SpawnResult`. Supports tmux/headless/interactive/rpc-keeper strategies. `SpawnResult` carries `spawnToken`, `keeperSockPath`, `code` (`SpawnFailureCode`). `buildSpawnEnv` takes optional `argv0`/`electronDeps`; when `argv0` is the Electron binary (`electronAsNodeRequired`) re-adds `ELECTRON_RUN_AS_NODE=1` that `resolver.buildSpawnEnv` strips (absent argv0 ⇒ byte-identical). `spawnWt`/`spawnHeadless` pass `piCmd[0]`; `spawnTmux`/`spawnWslTmux` untouched (shell token). See changes: spawn-correlation-token, add-rpc-stdin-dispatch-with-keeper-sidecar, spawn-failure-diagnostics, fix-nodescript-argv-electron-execpath-fallback.
