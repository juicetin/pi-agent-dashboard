# DOX — packages/shared/src/platform

Files in this directory. One row per source file.

| File | Purpose |
|------|---------|
| `binary-lookup.ts` | Unified binary resolution. Exports `ToolResolver` — `which`, `resolvePi`, `resolveTsx`, `resolveNode`, `resolveJiti`, `buildSpawnEnv`. Exports `whichSync`, `isAppImageSelfHit`, `MANAGED_PI_PACKAGES`, `JITI_PACKAGES`, `ResolveJitiOpts`, `AppImageSelfHitOpts`. Search order: managed bin → extraBinDirs → system PATH → login shell (macOS/Linux GUI). jiti hook resolution walks managed-pi → system-pi → caller anchor → `process.argv[1]`. Strips Electron env vars from spawn PATH. |
| `commands.ts` | Cross-platform OS-command primitives. `openBrowser(url,opts)` opens default browser (darwin `open`, win32 `start`, linux `xdg-open`). `isVirtualMachine(opts)` best-effort VM detect: darwin `sysctl -n hw.model`, linux `systemd-detect-virt`, win32 PowerShell `Get-CimInstance Win32_BIOS`+`Win32_ComputerSystem` via `spawnSync` (no wmic; wmic removed by default Win 11 22H2+). `parseVmProbeOutput(text)` pure regex helper `VMware\|VirtualBox\|VBOX\|Parallels\|Virtual Machine\|Hyper-V`. Injectable `platform`/`exec`/`asyncExec`/`spawnSync` opts for tests. See change: replace-wmic-with-powershell. |
| `detached-spawn.ts` | OS-aware detached-child spawn primitives. Exports `spawnDetached`, `waitForNoCrash`, `waitForReady` + option/result interfaces. Invariants: `detached: true` default, `shell: false` always, `stdio[0]="ignore"`, `windowsHide: true`. `logFd` routes both stdout+stderr to single fd. `detach:false` ties child to parent Job Object (Windows) / process group (POSIX). |
| `ensure-bundled-git.ts` | `ensureBundledGitOnPath()` + `resolveBundledGitDir()`. Prepends bundled git/sh dirs to PATH (`cmd`, `usr/bin`, `<libdir>/bin`). Sets `GIT_EXEC_PATH` + `SSL_CERT_FILE`. `<libdir>` arch-specific: `mingw64` (x64) / `clangarm64` (arm64). Shell = `usr/bin/sh.exe` (dugite-native ships NO `bash.exe`). Idempotent. Only module allowed to hardcode `resources/git` paths (`no-hardcoded-bundled-git-paths.test.ts`). See change: embed-git-bash-on-windows. |
| `ensure-windows-path.ts` | `ensureWindowsSystemPath(env, opts?)`: on win32 prepends to PATH (only when dir exists on disk + not already substring-present, case-insensitive): `C:\Windows\System32`, `C:\Windows`, `\System32\Wbem`, `\System32\WindowsPowerShell\v1.0`, `\System32\OpenSSH`, `%LOCALAPPDATA%\Microsoft\WindowsApps`. Idempotent. Non-win32 no-op. Shallow-clones `env`, never mutates `process.env`. Test seam via `{ platform, exists }` opts. Called from `ToolResolver.buildSpawnEnv` (binary-lookup.ts) so every spawned child inherits System32 even when parent PATH lacks it (Electron / minimal-env launchers). Re-exported by `packages/electron/src/lib/ensure-windows-path.ts` for back-compat. See change: fix-windows-path-system32-missing. Sibling pattern to `ensure-bundled-git.ts`. See change: embed-git-bash-on-windows. |
| `exec.ts` | Safe `child_process` wrappers forcing `windowsHide: true`. Exports `execSync`, `execFileSync`, `spawnSync`, `exec`, `execFile`, `spawn`, `execAsync`, `execFileAsync`, `buildSafeArgv`, `SafeArgv`. `buildSafeArgv` wraps `.cmd`/`.bat` + extensionless names in `cmd.exe /d /s /c` on Windows; native `.exe` + Unix spawn direct. Never `shell: true`. Re-exports child_process types. |
| `git-source.ts` | Process-lifetime git-source cache. `setWindowsGitSourceSetting`, `getActiveGitSource(which)`, `augmentEnvWithGitSource(env, which)`, `getGitSourceReadout(which)` (Windows-only, cached), `invalidateGitSourceCache`. `which` injected to avoid binary-lookup cycle. See change: embed-git-bash-on-windows. |
| `git.ts` | Recipe-based git API. Exports `Recipe` consts `GIT_IS_REPO`, `GIT_CURRENT_BRANCH`, `GIT_HEAD_SHA`, `GIT_REMOTE_URL`, `GIT_COMMON_DIR`, `GIT_TOPLEVEL`, `GIT_DIFF`, `GIT_STATUS_PORCELAIN`, `GH_PR_NUMBER` + `GIT_RECIPES`. Typed funcs `isGitRepo`/`currentBranch`/`headSha`/`remoteUrl`/`commonDir`/`toplevel`/`diff`/`statusPorcelain`/`prNumber` return `Result<T>`. `*Or` variants swallow errors → fallback. Thin wrappers over `run()`. |
| `index.ts` | Barrel re-export of platform modules. Exports `binary-lookup`, `process`, `process-scan`, `shell`, `commands`, `exec`, `runner`, `detached-spawn`, `spawn-mechanism`, `process-identify`, `subprocess-adapter`, `node-spawn` flat; `git`, `openspec`, `npm`, `paths` as namespaces. |
| `managed-node-path.ts` | `getManagedNodeBinDir(managedDir, platform?)` + `prependManagedNodeToPath(env, managedDir, platform?)`: shallow-clones `env`, prepends `<managedDir>/node/` (Win) or `<managedDir>/node/bin` (Unix) to `PATH`. No-op when runtime absent. Never mutates `process.env`. Consumed by every spawned-child env builder (pi-session, pi-core-updater, headless, server-launcher) so managed `node`/`npm`/`npx` resolves first. See change: embed-managed-node-runtime. |
| `node-spawn.ts` | Canonical `node --import <loader> <entry>` argv builder. Exports `spawnNodeScript`, `buildNodeImportArgvParts`, `isTsxLoader`, `isJitiLoader`, `shouldUrlWrapEntry`, `toFileUrl`, `SpawnNodeScriptOptions`. URL-wraps loader always; entry wrap per `shouldUrlWrapEntry` (tsx+jiti reject `file://` entries; Windows default Node wraps). Single source for `--import` argv shape; lint `no-raw-node-import` enforces. |
| `npm.ts` | Recipe-based npm CLI API. Exports `Recipe` consts `NPM_ROOT_GLOBAL`, `NPM_OUTDATED`, `NPM_OUTDATED_GLOBAL`, `NPM_INSTALL`, `NPM_INSTALL_GLOBAL`, `NPM_VIEW_VERSION` + `NPM_RECIPES`. Typed funcs `rootGlobal`/`outdated`/`outdatedGlobal`/`install`/`installGlobal`/`viewVersion` return `Result<T>`; `*Or` variants swallow errors. Long install timeout 120s; outdated tolerates exit 1. |
| `openspec.ts` | Recipe-based OpenSpec tool module. Adds `OPENSPEC_CONFIG_PROFILE` + `OPENSPEC_UPDATE` recipes, `configProfile()`/`update()` runners, `writeOpenSpecConfigFile()` (atomic tmp+rename), `workflowSetSignature()`, `openSpecConfigFilePath()`. Adds `configListAsync` + `configListOrAsync` (async, non-blocking sibling of `configList`/`configListOr`, uses `runAsync`). Re-exports `CORE_WORKFLOWS`/`EXPANDED_WORKFLOWS` from types. See changes: add-openspec-profile-settings, fix-openspec-profile-load-race. |
| `paths.ts` | Isomorphic OS-aware path primitives (no `node:path`; browser-safe). Exports `normalizePath`, `samePath`, `parsePathInput`, `withTrailingSep`, `joinForDisplay`, `isFilesystemRoot`. Windows drive-letter + UNC root aware; case-insensitive on win32/darwin, case-sensitive on linux. Every helper takes injectable `platform` param. Used by pin storage, session grouping, path-picker UI. |
| `process-identify.ts` | Process identification by command-line marker. Exports `isPiCommandLine`, `findPidByMarker`, `isProcessLikePi`, `ProcessIdentifyOpts`. Unix: `ps -eo pid,command` + sentinel match; Windows stubs `[]`/`true` (delegates to headlessPidRegistry). Injectable `platform`/`exec` for tests. |
| `process-scan.ts` | Cross-platform process enumeration. Exports `parseEtime` (ps `etime` → ms), `isProcessRunning` (win32 tasklist, unix pgrep), `ProcessScanOpts`. Injectable `platform`/`exec`. Best-effort: failures return false/0. |
| `process.ts` | Cross-platform process primitives. Exports `parseNetstatListeners`, `findPortHolders`, `isProcessAlive`, `killProcess`, `killPidWithGroup`, `ProcessOpts`, `KillProcessResult`. Unix: `lsof`/SIGTERM→SIGKILL; Windows: `netstat -ano`/`taskkill /F /T`. Injectable `platform`/`exec`/`kill`. |
| `runner.ts` | Recipe execution engine. Exports `run`, `runAsync`, `unwrap`, `buildSpawnEnvForArgv`, `electronAsNodeRequired`, `resetResolverCache`, types `Recipe`, `RunCtx`, `Result`, `ExecError`. Resolves argv[0] via tool registry (executor form) → `ToolResolver.which` fallback; routes through `buildSafeArgv`; enforces timeout + tolerated exit codes; normalizes errors to `Result<T>` (never throws). `electronAsNodeRequired(argv0, deps?)` = shared predicate: `Boolean(electronVersion) && argv0 === execPath`; `buildSpawnEnvForArgv` sets `ELECTRON_RUN_AS_NODE=1` via it when node-wrapping Electron execPath. Predicate reused by `process-manager.buildSpawnEnv` + `keeper-manager` so every executor spawn site agrees. See change: fix-nodescript-argv-electron-execpath-fallback. |
| `select-git-source.ts` | `selectGitSource()` tri-state `windowsGitSource` decision (`auto`/`host`/`bundled`) → `"host"`\|`"bundled"`. Pure, injectable `which`/`platform`. Non-win32 → host. `auto` needs git+bash on host PATH atomically else bundled. See change: embed-git-bash-on-windows. |
| `shell.ts` | Cross-platform shell + terminal env primitives. Exports `detectShell` (win32 COMSPEC/powershell, unix SHELL/bash), `getTerminalEnvHints` (win32 sets `TERM=cygwin`), `ShellOpts`. Injectable `platform`/`env`. |
| `spawn-mechanism.ts` | Session-spawn mechanism selection. Exports `selectMechanism`, `buildWtArgs`, `sessionFlagsToArgv`, types `SpawnMechanism`, `UserSpawnStrategy`, `SpawnMechanismContext`, `WtArgsOptions`, `SessionFlags`. Mechanisms: tmux / wt / wsl-tmux / headless. `selectMechanism` pure; electronMode or userStrategy "headless" forces headless. `buildWtArgs` argv form reuses MRU WT window. `sessionFlagsToArgv` appends `--session`/`--fork`/`--model`. |
| `subprocess-adapter.ts` | Strategy-pattern OS-aware subprocess adapter. Exports `SubprocessAdapter` interface, `createSubprocessAdapter`, `getDefaultSubprocessAdapter`, `_resetDefaultSubprocessAdapter`. Windows adapter routes `.cmd`/extensionless via `buildSafeArgv` (`cmd.exe /d /s /c`); Unix adapter direct spawn `shell:false`. Lazy process-wide singleton; injectable for tests. |
