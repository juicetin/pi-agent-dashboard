# Parity harness (task 10.2)

Docker-based check that the TypeScript port produces SRT **byte-identical** to the
original Python `video-transcription` skill. Dev/QA only — not published (`parity/`
is excluded from the package `files` whitelist).

## Run

From the package dir (`packages/video-transcription`):

```bash
# Offline converter parity only (deterministic, no API key, no network):
./parity/run.sh
# or, without Docker:
npm run parity

# Offline + live end-to-end (needs a real key, a sample, and ffmpeg):
SONIOX_API_KEY=sk-... PARITY_SAMPLE=/abs/path/clip.m4a ./parity/run.sh
```

## Two tiers

1. **Offline converter parity — always runs, deterministic.**
   Soniox feeds identical tokens to both implementations, so the only place they
   can diverge is token→SRT conversion. This tier runs the TS `tokensToSrt` over
   `fixtures/tokens.json` and diffs against `fixtures/tokens.expected.srt`. A
   mismatch exits non-zero. No API key or network required.

2. **Live end-to-end — guarded.**
   Activates only when `SONIOX_API_KEY` **and** `PARITY_SAMPLE` are set. Runs the
   full `pi-transcribe` pipeline (audio extraction via ffmpeg → Soniox → SRT) on
   the sample inside the container and asserts well-formed, non-empty SRT.
   Skipped otherwise, so the check never blocks CI. The sample is mounted
   read-only and copied to a writable temp dir before transcription.

## Reference SRT provenance

`fixtures/tokens.expected.srt` is generated by the **real Python converter** from
the original skill, over `fixtures/tokens.json`:

```bash
PY=~/.pi/agent/skills/video-transcription/venv/bin/python3
"$PY" - <<'EOF' > parity/fixtures/tokens.expected.srt
import sys, json
sys.path.insert(0, "/Users/<you>/.pi/agent/skills/video-transcription/scripts")
from transcription import TranscriptionService
svc = TranscriptionService("dummy")           # key unused for conversion
data = json.load(open("parity/fixtures/tokens.json"))
sys.stdout.write(svc._convert_to_srt(data))
EOF
```

Regenerate this reference only if `fixtures/tokens.json` changes. The fixture
intentionally exercises: multi-speaker segments, a speaker-change boundary, the
5000 ms max-segment split, and a blank-text token skip.

The deterministic segmentation/format parity is also enforced by the vitest
golden test (`src/__tests__/srt.test.ts`, task 3.4); this harness adds the
Docker + live-sample confirmation.
