# DOX — qa/packer

Files in this directory. One row per file. Non-source area. See change: migrate-file-index-to-agents-tree. See change: fold-oversized-agents-directories.

| File | Purpose |
|------|---------|
| `http/autounattend.xml` | Windows 11 unattended install answer file. `windowsPE` pass: EFI/MSR/Primary partition layout, Win11 hardware-check bypass via `LabConfig` reg keys (`BypassTPMCheck`, `BypassSecureBootCheck`, `BypassRAMCheck`, `BypassCPUCheck`, `BypassStorageCheck`), `/IMAGE/NAME=Windows 11 Enterprise Evaluation` edition disambiguation. `oobeSystem` pass: creates `qa`/`qa` admin account, auto-logon, `FirstLogonCommands` install OpenSSH.Server capability, set sshd auto-start, open firewall TCP/22, disable firewall profiles, set DefaultShell to PowerShell. |
| `http/meta-data` | cloud-init NoCloud datasource meta-data file (empty). Required presence for `ds=nocloud-net` boot; meta-data populated by instance-id/local-hostname if needed. |
| `http/user-data` | Ubuntu cloud-init autoinstall config (`#cloud-config`). Sets hostname `qa-ubuntu`, user `qa` (SHA-512 crypt password "qa"), installs openssh-server/curl/ca-certificates, LVM storage layout, authorized-keys (`REPLACE_WITH_YOUR_PUBLIC_KEY` placeholder), password-less sudo via `/target/etc/sudoers.d/qa`. Served via `http_directory = "http"` + `autoinstall ds=nocloud-net` boot command. |
| `macos-arm.pkr.hcl` | Packer template for macOS ARM (M1). `vmware-vmx` builder (`source.vmware-vmx.macos-arm`) starts from existing `var.source_vmx` VM (no ISO unattended install). Vars: `source_vmx`, `vm_name=macos-14-arm`, `ssh_username=qa`, `ssh_password=qa`, `cpus=4`, `memory=8192`, `ssh_timeout=15m`. Provisions via `provision-common.sh` + `provision-macos.sh`. Output `../output/${var.vm_name}`. |
| `macos-x86.pkr.hcl` | Packer template for macOS x86. `vmware-vmx` builder (`source.vmware-vmx.macos`) from existing `var.source_vmx`. Vars: `source_vmx`, `vm_name=macos-14-x86`, `ssh_username=qa`, `ssh_password=qa`, `cpus=4`, `memory=8192`, headless. Provisions `provision-common.sh` + `provision-macos.sh`. Output `../output/${var.vm_name}`. |
| `scripts/provision-common.sh` | Shared provisioner: installs nvm v0.40.1 (`curl raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh`), Node.js LTS (`nvm install --lts`, `nvm alias default lts/*`), appends NVM_DIR loader to `~/.bashrc`. Called by `provision-linux.sh` and `provision-macos.sh`. |
| `scripts/provision-linux.sh` | Ubuntu provisioner. `apt update/upgrade`, installs `build-essential curl git openssh-server python3 ca-certificates xvfb libnss3 libgtk-3-0 libgbm1 libasound2` (xvfb + Electron shared libs for `08-electron-real-launch.sh`), enables ssh, runs `provision-common.sh`, cleans apt cache. |
| `scripts/provision-macos.sh` | macOS provisioner. Installs Xcode Command Line Tools via `softwareupdate` (headless-friendly touch trick), Homebrew (`NONINTERACTIVE=1`), `brew install git`, runs `provision-common.sh`. |
| `scripts/provision-windows.ps1` | Windows provisioner. Installs Chocolatey, git, nvm-windows, Node.js LTS, VS Build Tools 2022 (`VCTools` workload for node-pty native compile), OpenSSH.Server capability + sshd auto-start + DefaultShell=PowerShell. `Update-Environment` helper re-imports Machine+User env block into Process scope (Chocolatey PATH staleness workaround). |
| `ubuntu-arm.pkr.hcl` | Packer template for Ubuntu ARM. `vmware-iso` builder (`source.vmware-iso.ubuntu-arm`), `guest_os=arm-ubuntu-64`, `cpus=2`, `memory=4096`, `disk_size=40960`, `http_directory=http`, `autoinstall ds=nocloud-net` boot command. Provisions `provision-common.sh` + `provision-linux.sh`. Output `../output/${var.vm_name}`. |
| `ubuntu-x86.pkr.hcl` | Packer template for Ubuntu x86. `vmware-iso` builder (`source.vmware-iso.ubuntu`), `guest_os=ubuntu-64`, `cpus=2`, `memory=4096`, `disk_size=40960`, `http_directory=http`, `autoinstall ds=nocloud-net` boot command. Provisions `provision-common.sh` + `provision-linux.sh`. Output `../output/${var.vm_name}`. |

| `windows.pkr.hcl` | Packer vmware-iso template builds Windows 11 base image. vmx_data enforces UEFI + Secure Boot + vTPM 2.0 (`firmware=efi`, `tpm.present=TRUE`), `disk_adapter_type=lsisas1068`, `network_adapter_type=e1000e`, `vhv.enable=TRUE`.
