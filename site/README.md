# pi-dashboard marketing site

Public-facing site served at https://blackbelttechnology.github.io/pi-agent-dashboard/
(later: https://pi-dashboard.dev once the domain is acquired).

## Stack

- **Astro 5** with static output
- **Tailwind CSS 3** (via `@astrojs/tailwind`, base styles disabled вЂ” we ship our own in `src/styles/global.css`)
- **MDX** (via `@astrojs/mdx`) for long-form copy
- **React 18** islands вЂ” only the hero animation and install tabs are hydrated
- **motion-one** for the hero storytelling animation
- **Playwright** for the screenshot pipeline

## Local development

```bash
cd site
npm install
npm run dev            # starts Astro dev server
npm run build          # produces site/dist/
npm run preview        # serves the built site locally
```

## Screenshot pipeline

The site's screenshots under `public/screenshots/` are generated by a scripted
Playwright pipeline that boots a temporary dashboard server with seeded demo
data, visits every feature panel at desktop and mobile viewports, and writes
PNGs back into the site.

```bash
# From the repo root:
npm run screenshots
```

See `scripts/screenshots/README.md` for details, troubleshooting, and when to
re-run (typically: whenever a UI change is visible in the bento grid).

Generated PNGs are **committed to git** вЂ” we don't regenerate on every CI run.

## Deploy

Pushes to `main` that touch `site/**` or `.github/workflows/deploy-site.yml`
trigger the deploy workflow, which builds the site and publishes via
`actions/deploy-pages@v4`. No `gh-pages` branch is used.

First-time setup (one-time, by a repo admin):

1. Merge this change's PR so `site/` and the workflow exist on `main`.
2. Open repo **Settings в†’ Pages**.
3. Set **Source** to **GitHub Actions**.
4. Trigger `Deploy Site` workflow manually (or push any change under `site/`).
5. Confirm the deployed URL renders.

## Custom domain (future: pi-dashboard.dev)

When the domain is acquired:

1. Create `site/public/CNAME` with a single line: `pi-dashboard.dev`.
2. Update `astro.config.mjs`:
   - `site: "https://pi-dashboard.dev"`
   - `base: "/"`
3. Configure DNS records per
   [GitHub Pages custom domain docs](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site).
4. Enable **Enforce HTTPS** in repo Settings в†’ Pages.
5. Regenerate screenshots so any absolute URL references match the new host
   (rare; most URLs are relative).
